<h1>SQL Data Quality Project</h1>
<p>
    This project explores a relational database of a bicycle retail and distribution company using Microsoft SQL Server.
	The goal of the project is apply data validation, reconciliation, anomaly detection, and data quality techniques.
</p>

## Project Structure

- `database/FunData.mdb` - original Microsoft Access database
- `database/FunData.xlsx` - exported tables from database
- `sql` - database setup scripts
- `screenshots` - screenshots

## Database Structure

- Customers
- Products
- Product Categories
- Suppliers
- Purchases
- Orders
- Credits
- Employees
- Financial Records

## Tools Used

- SQL Server Management Studio (SSMS)
- Microsoft Access
- Microsoft Excel
- Visual Studio Code
- GitHub

## Database Migration

- The original database was provided as a Microsoft Access (.mdb) file.
- The Access database was exported into Microsoft SQL Server.
- Data was extracted into Excel files for validation.
- SQL queries were used to explore and validate the data.

This process provided practical experience with database migration, data preparation, and SQL-based analysis.

## Project Objectives

- Explore the database structure
- Understand relationships between tables
- Perform data quality checks
- Identify missing and inconsistent records
- Detect anomalies in operational data
- Perform reconciliation between related datasets
- Analyse inventory and purchasing risks
